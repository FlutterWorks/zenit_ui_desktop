# Button

A button is an interactive component which, when clicked, performs some action.

## Primary buttons

Primary buttons are colorful and stand out. They indicate the most important call to action available at any given point.

<Story id="components-button--primary" />

Primary buttons are best used once per screen for the most prominent action, such as submitting a form or logging into a website.
It should be for a positive action, even if this action is destructive: for example, use a primary button for confirming deletions.

## Secondary buttons

These less obvious buttons don't stand out, so they're useful for secondary actions, such as canceling an action.

<Story id="components-button--secondary" />

Since primary buttons contain the main action in a group, secondary buttons are best used next to primary buttons for alternative actions.

### Implementation details

```css
/* CSS */

border-radius: 24px;

  background: color.lighten(var(--secondary-color) 16%);
  /* darken in light mode, lighten in dark mode */
}
:active {
  background: color.lighten(var(--secondary-color) 32%);
}
```

## Transparent buttons

<Story id="components-button--tertiary" />

Also called "tertiary" or "text" buttons, this type is best used in app bars and toolbars, rather than in dialogs. It does not have a background or outline, and is identified by expectation alone.
If you use transparent buttons in a context where it isn't obvious, you will encounter accessibility issues, so be careful how you use these.

When the mouse is resting over a transparent button, it brightens or dims the background area just as it would if there were a background color set for the button.
